# Design contract

Penpot owns approved visual decisions. Vue owns runtime behavior.

The workflow has three Penpot areas:

- `Explorations / Vue in Motion` for alternatives and UX review.
- `Approved / ...` for the accepted website, design system, and component masters.
- `Code snapshot / ...` for generated boards that reflect the current Vue implementation.

Only `Code snapshot / ...` boards may be deleted and regenerated by automation.

After a design is approved:

1. Run `bun run design:contract`.
2. Review the resulting `penpot-design-contract.json` diff.
3. Implement the affected Vue components and tokens.
4. Run `bun run design:sync` to rebuild the code snapshot.
5. Compare Approved and Code snapshot at matching viewport sizes.

`component-code-map.json` maps approved Penpot components to their Vue implementation and
lists the states that UX review should cover.
